### vercel-react-best-practices

When writing or reviewing React/Next.js code:
- **Waterfalls (CRITICAL)**: Use `Promise.all()` for independent fetches; defer `await` to usage point; use Suspense boundaries
- **Bundle (CRITICAL)**: Import directly from source files (not barrel files); use `next/dynamic` for heavy components; lazy-load non-critical libs
- **Server (HIGH)**: Always auth inside Server Actions; use `React.cache()` for per-request deduplication; parallelize RSC fetches via component composition
- **Re-renders (MEDIUM)**: Derive state during render (no effects for derived values); use functional setState; narrow effect dependencies to primitives
- **Rendering (MEDIUM)**: Use `content-visibility: auto` for long lists; wrap SVG in div for animations; explicit ternary for conditional rendering
- **JS perf (LOW-MEDIUM)**: Build index Maps for repeated lookups; combine array iterations; early returns; use `toSorted()` not `sort()`
